    **1. Add routes** to `lib/astraplex_web/router.ex`:

      - `live "/dm/:id", ConversationLive, :show`

    **2. Create ConversationLive** (`lib/astraplex_web/live/conversation_live.ex`):
    Mirror ChannelLive structure but handle two mount modes:

    **Mount for `:show` (existing conversation):**
    - Extract `id` from params, load conversation via `Ash.get!(Conversation, id, actor: current_user, load: [:members])`
    - Load messages: query Message where conversation_id == id, sort by inserted_at asc, limit 50, load [:sender]
    - Load sidebar data: channels (reuse load_sidebar_channels pattern) + conversations (new load_sidebar_conversations helper)
    - Subscribe to PubSub: `"conversation:messages:#{id}"` for message delivery, `"user:conversations:#{current_user.id}"` for sidebar updates, `"membership:changed:#{current_user.id}"` for channel sidebar updates
    - Assign: conversation, conversations (sidebar), channels (sidebar), current_conversation_id, has_messages?, stream messages

    **Mount for `:new` (lazy creation -- no conversation persisted yet):**
    - Two sub-modes based on query params:
      a) No `members` param: show user picker view for selecting participants
      b) Has `members` param: show chat view ready for first message
    - For (a): load all active users, initialize empty selected_ids list, show picker UI
    - For (b): parse member IDs from comma-separated param, compute member_hash (sort [current_user.id | member_ids] and join with ","), check if conversation already exists via `find_by_member_hash` action. If exists: redirect to `/dm/#{existing.id}` via `push_navigate`. If not: load the selected members for display, set assigns: conversation = nil, members = loaded users, member_ids = parsed IDs
    - Subscribe to PubSub for sidebar updates only (no conversation topic yet)

    **Render:**
    - Role-based shell (admin_shell/staff_shell) wrapping content, same pattern as ChannelLive
    - Pass conversations and current_conversation_id to shell (alongside existing channels/current_channel_id)
    - Breadcrumb: [{"Home", "/"}, {conversation_display_name, nil}]
    - Two render paths:
      a) User picker mode (@live_action == :new and no members selected): show user_picker component with "Start Conversation" button
      b) Chat mode (existing conversation or members selected): show conversation_content with chat_layout

    **conversation_content function component:**
    - Use `Layouts.chat_layout` with title = conversation_display_name(conversation_or_members, current_user)
    - Title action: people icon button that opens member list drawer (read-only, using existing member_list component)
    - Message area: stream-based rendering with message_bubble (same as ChannelLive)
    - Empty state when no messages
    - Input: form with phx-submit="send_message", ClearInput hook, text input + submit button
    - Member list drawer: DaisyUI drawer with member_list component (removable: false)

    **conversation_display_name/2 helper:**
    - Takes conversation (with members loaded) or list of members + current_user
    - Filter out current_user from members
    - 1 other member: show their email
    - 2 other members: "alice, bob" (email prefix before @)
    - 3+ other members: "alice, bob +N"

    **handle_event "send_message":**
    - If @conversation is nil (lazy creation mode):
      1. Call `Ash.create!(Conversation, %{}, action: :create_with_message, actor: current_user, arguments: %{member_ids: @member_ids, body: body})`
      2. Broadcast to each member's user topic: `AstraplexWeb.Endpoint.broadcast("user:conversations:#{member_id}", "conversation_created", %{})` for each member
      3. `push_patch` to `/dm/#{conversation.id}` to update URL
      4. Load the created conversation with members, subscribe to its PubSub topic
      5. Stream the first message
    - If @conversation exists:
      1. `Ash.create!(Message, %{body: body, conversation_id: @conversation.id}, action: :send_conversation_message, actor: current_user)`
      2. Touch the conversation's updated_at: `Ash.update!(@conversation, %{}, action: :touch, actor: current_user)`
      3. Broadcast sidebar update to all members: `AstraplexWeb.Endpoint.broadcast("user:conversations:#{member.id}", "conversation_updated", %{})` for each member

    **handle_event for user picker** (in :new mode without members):
    - "toggle_member" -- add/remove user ID from selected_ids list
    - "search_members" -- filter available users by email (client-side, same as channel member picker)
    - "start_conversation" -- compute member_hash from selected IDs + current_user, check existence, navigate to `/dm/new?members=ids` or `/dm/{existing_id}`

    **handle_info for PubSub messages:**
    - `"conversation:messages:" <> _` -- load sender on message, stream_insert at -1, set has_messages? true (mirror ChannelLive pattern)
    - `"user:conversations:" <> _` -- reload sidebar conversations
    - `"membership:changed:" <> _` -- reload sidebar channels (existing pattern)

    **load_sidebar_conversations/1 helper:**
    - `Ash.read!(Conversation, action: :list_for_user, actor: actor)`
    - Map to `%{id: to_string(c.id), label: conversation_display_name(c, actor), url: ~p"/dm/#{c.id}"}`

    **IMPORTANT scroll behavior note:** The existing chat_layout with `flex flex-col justify-end min-h-full` on the message container already makes messages start at top and pin to bottom when full. The ScrollBottom hook handles auto-scroll on new messages. This should work correctly out of the box -- verify during implementation.
