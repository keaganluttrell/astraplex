## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added layout option to live_sessions in router**
- **Found during:** Task 2 (shell layout tests)
- **Issue:** Live sessions did not specify a `layout:` option, so the `app/1` function was never called by Phoenix. LiveView content rendered directly without any app layout wrapper.
- **Fix:** Added `layout: {AstraplexWeb.Layouts, :app}` to both `:authenticated` and `:admin` live_sessions in router.ex
- **Files modified:** lib/astraplex_web/router.ex
- **Verification:** All layout tests pass, shell renders around LiveView content
- **Committed in:** 9de9d7d (Task 2 commit)

**2. [Rule 1 - Bug] Made bridge app/1 role-aware instead of always staff_shell**
- **Found during:** Task 2 (admin shell tests)
- **Issue:** Bridge `app/1` always delegated to `staff_shell`, hiding Admin link for admin users
- **Fix:** Added cond that checks `current_user.role == :admin` to select admin_shell vs staff_shell
- **Files modified:** lib/astraplex_web/components/layouts.ex
- **Verification:** Admin layout test passes -- admin users see Admin link in sidebar
- **Committed in:** 9de9d7d (Task 2 commit)
