    - Test: Admin can change a user's role via the user list
    - Test: Admin can deactivate an active user (confirmation modal appears)
    - Test: Deactivated users show "Deactivated" badge in the table
    - Test: Admin can reactivate a deactivated user
    - Test: Staff user cannot access /admin/users (redirected)
  </behavior>
  <action>
    **UserListLive (lib/astraplex_web/live/admin/user_list_live.ex):**
    Single LiveView handling both :index and :new live_actions.

    Mount: Load all users via `Ash.read!(Astraplex.Accounts.User, actor: current_user)`. Assign to `@users`.

    Render (:index live_action):
    - Page title: "User Management"
    - "New User" button (link to /admin/users/new or opens modal)
    - Table with columns: Email, Role (badge: "Admin" primary / "Staff" neutral), Status (badge: "Active" success / "Deactivated" error), Actions
    - Actions column per user:
      - "Change Role" button — calls handle_event that invokes :update_role action (toggles admin<->staff)
      - "Deactivate" button (for active users) — opens DaisyUI modal with confirmation text: "Are you sure you want to deactivate [email]? They will be logged out immediately." Per locked decision.
      - "Reactivate" button (for deactivated users) — directly calls :reactivate action
    - Use DaisyUIComponents for table, badges, buttons, modal
    - Keep LiveView under 150 lines per CLAUDE.md. If approaching limit, extract table row rendering into a function component in the same file.

    Render (:new live_action):
    - Modal or inline form with fields: email (text input), password (password input), role (select dropdown with Admin/Staff options)
    - Use `AshPhoenix.Form.for_action(Astraplex.Accounts.User, :create_user, actor: current_user)` for the form
    - On submit: validate form, if valid submit via `AshPhoenix.Form.submit/2`
    - On success: redirect to :index, flash success message
    - On error: show form errors

    Handle events:
    - `"deactivate"` with user id — invoke Accounts deactivate action with admin actor
    - `"reactivate"` with user id — invoke Accounts reactivate action with admin actor
    - `"change_role"` with user id — toggle role (staff->admin or admin->staff) via update_role action
    - After each action, reload users list

    All data access through Ash actions with `actor: socket.assigns.current_user`. No raw Ecto. No business logic in LiveView (just delegates to Ash actions).

    Write tests FIRST in test/astraplex_web/live/admin/user_list_live_test.exs using ConnCase with `register_and_log_in_user(context, role: :admin)`.
  </action>
