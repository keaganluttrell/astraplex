    **AuthController (lib/astraplex_web/controllers/auth_controller.ex):**
    Implement using `AshAuthentication.Phoenix.Controller` (or `use AstraplexWeb, :controller` with manual session clearing). The controller handles:
    - `sign_out/2` — clears session, redirects to "/sign-in"
    - Sign-in success/failure callbacks if required by AshAuthentication.Phoenix

    Use `AshAuthentication.Phoenix.Plug` helpers for session management. The `sign_out` action should use `AshAuthentication.Phoenix.Plug.sign_out/2` or equivalent to clear the session properly.

    **LiveAuth (lib/astraplex_web/live/live_auth.ex):**
    Follow Pattern 5 from research:
    - `on_mount(:require_authenticated_user, ...)` — if `socket.assigns[:current_user]` is nil, halt and redirect to "/sign-in". Also check that user status is :active (handles deactivation while session exists).
    - `on_mount(:require_admin, ...)` — if current_user is nil or current_user.role != :admin, halt and redirect to "/". This ensures staff cannot access admin LiveViews.
    - `on_mount(:redirect_if_authenticated, ...)` — if current_user exists, redirect to "/" (prevents logged-in users from seeing sign-in page).

    **Router (lib/astraplex_web/router.ex):**
    Major rewrite. Add `use AshAuthentication.Phoenix.Router` at top.

    Add `:load_from_session` plug to browser pipeline (per research pitfall 6 — this is critical for current_user to be available in LiveViews).

    Structure:
    ```
    # Unauthenticated routes (sign-in page)
    scope "/", AstraplexWeb do
      pipe_through :browser

      sign_in_route(path: "/sign-in", live_view: AstraplexWeb.AuthLive.SignInLive)
      sign_out_route AuthController
      auth_routes AuthController, Astraplex.Accounts.User, path: "/auth"
    end

    # Authenticated routes
    ash_authentication_live_session :authenticated,
      on_mount: [{AstraplexWeb.LiveAuth, :require_authenticated_user}] do
      scope "/", AstraplexWeb do
        pipe_through :browser
        live "/", DashboardLive, :index
      end
    end

    # Admin routes
    ash_authentication_live_session :admin,
      on_mount: [{AstraplexWeb.LiveAuth, :require_admin}] do
      scope "/admin", AstraplexWeb.Admin do
        pipe_through :browser
        live "/users", UserListLive, :index
        live "/users/new", UserListLive, :new
      end
    end
    ```

    Remove the old `get "/", PageController, :home` route. Keep the /mcp scope and dev routes intact.

    Write LiveView connection tests FIRST in test/astraplex_web/live/auth_live_test.exs, then implement.
