**Router updates:**


**ChannelLive (`lib/astraplex_web/live/channel_live.ex`):**

Mount:
- Extract `channel_id` from params
- Load channel: `Ash.get!(Astraplex.Messaging.Channel, channel_id, actor: current_user)`
- Load messages: `Ash.read!(Astraplex.Messaging.Message, actor: current_user)` filtered by channel_id, sorted by inserted_at asc, with sender loaded. Use an inline filter or a custom read action. Load last 50 messages initially.
- Load user's channels for sidebar: `Ash.read!(Astraplex.Messaging.Channel, action: :list_for_user, actor: current_user)`
- Format channels for sidebar: `Enum.map(channels, fn c -> %{id: c.id, label: "#" <> to_string(c.name), url: ~p"/channels/#{c.id}"} end)`
- Subscribe to PubSub (if connected):
  - `AstraplexWeb.Endpoint.subscribe("channel:messages:#{channel_id}")` for new messages
  - `AstraplexWeb.Endpoint.subscribe("membership:changed:#{current_user.id}")` for sidebar updates
- Initialize message stream: `stream(socket, :messages, messages)`
- Assign: channel, channels (for sidebar), current_channel_id, message_form (empty form for input)

Render:
- Use the role-based shell (admin_shell or staff_shell based on current_user.role, like DashboardLive pattern)
- Pass `channels={@channels}` and `current_channel_id={@current_channel_id}` to the shell
- Inside shell, use `chat_layout` component with `title={"#" <> to_string(@channel.name)}`
- Message area: render stream of messages using `message_bubble` component (from messaging components, or inline if Plan 02 hasn't run yet -- create a simple inline version)
- If no messages, show `empty_state` with icon="hero-chat-bubble-left-right", title="No messages yet", description="Be the first to say something!"
- Input slot: form with text input and send button, `phx-submit="send_message"`
- If channel is archived, show a banner "This channel has been archived" instead of the input form

Handle events:
- `"send_message"`: extract body from params, call `Ash.create!(Message, %{body: body, channel_id: channel.id}, action: :send_message, actor: current_user)`, clear the message form. Do NOT manually insert into stream -- PubSub notification handles it.
- Actually, the sender's own message also arrives via PubSub, so just clear the form.

Handle info (PubSub):
- `%{topic: "channel:messages:" <> _, payload: %Ash.Notifier.Notification{data: message}}`:
  Load sender on message: `Ash.load!(message, [:sender], actor: current_user)`
  Insert into stream: `stream_insert(socket, :messages, message, at: -1)`
- `%{topic: "membership:changed:" <> _}`:
  Re-query channels: reload list_for_user, reformat for sidebar, update assigns
  This handles both being added to and removed from channels.

**DashboardLive update:**
Update the existing DashboardLive to also load channels for the sidebar:
- In mount, add: load channels via `:list_for_user`, format for sidebar
- Pass `channels={@channels}` and `current_channel_id={nil}` to the shell
- Subscribe to `"membership:changed:#{current_user.id}"` for sidebar updates
- Handle info for membership changes to refresh sidebar

**Message bubble fallback:**
If the messaging components module from Plan 02 doesn't exist yet (plans run in parallel), create an inline `message_bubble` function component directly in ChannelLive or as a simple private function. It needs: sender avatar, sender email, timestamp, message body. Use DaisyUI chat-bubble classes for alignment (own messages right, others left).

Keep ChannelLive under 150 lines per CLAUDE.md LiveView rules. Extract helper functions as needed.
