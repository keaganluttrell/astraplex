Create the Messaging Ash domain and three resources following the patterns from RESEARCH.md.

**1. Domain module** (`lib/astraplex/messaging/messaging.ex`):
- `use Ash.Domain, otp_app: :astraplex`
- Register all three resources: Channel, Membership, Message

**2. Channel resource** (`lib/astraplex/messaging/channel.ex`):
- `use Ash.Resource` with `data_layer: AshPostgres.DataLayer, authorizers: [Ash.Policy.Authorizer], domain: Astraplex.Messaging`
- Attributes: `uuid_primary_key(:id)`, `name` (`:ci_string`, allow_nil?: false, public?: true, constraints: max_length 80, trim?: true), `description` (`:string`, allow_nil?: true, public?: true, constraints: max_length 500), `status` (`:atom`, constraints: one_of [:active, :archived], default: :active, allow_nil?: false, public?: true), `create_timestamp(:inserted_at)`, `update_timestamp(:updated_at)`
- Identity: `:unique_name` on `[:name]`
- Relationships: `has_many :memberships`, `has_many :messages`, `many_to_many :members` through Membership (source_attribute_on_join_resource: :channel_id, destination_attribute_on_join_resource: :user_id)
- Actions: `defaults [:read]`, `:create` (accept [:name, :description]), `:update` (accept [:name, :description]), `:archive` (accept [], change set_attribute(:status, :archived)), `:list_for_user` (read action, filter `expr(status == :active and exists(memberships, user_id == ^actor(:id)))`, prepare `build(sort: [name: :asc])`)
- Policies: create/update/archive require `expr(^actor(:role) == :admin)`. Read allows admin OR `relates_to_actor_via(:members)`. `list_for_user` allows always (filter handles scoping).
- Postgres: table "channels", repo Astraplex.Repo

**3. Membership resource** (`lib/astraplex/messaging/membership.ex`):
- Attributes: `uuid_primary_key(:id)`, `create_timestamp(:inserted_at)`
- Relationships: `belongs_to :channel` (allow_nil?: false), `belongs_to :user, Astraplex.Accounts.User` (allow_nil?: false)
- Identity: `:unique_membership` on `[:channel_id, :user_id]`
- Actions: `defaults [:read, :destroy]`, `:create` (accept [:channel_id, :user_id])
- Policies: create/destroy require admin. Read allows admin OR `expr(exists(channel.memberships, user_id == ^actor(:id)))`.
- PubSub notifier: module `AstraplexWeb.Endpoint`, prefix "membership", publish :create ["changed", :user_id], publish :destroy ["changed", :user_id]
- Postgres: table "memberships", repo Astraplex.Repo

**4. Message resource** (`lib/astraplex/messaging/message.ex`):
- Attributes: `uuid_primary_key(:id)`, `body` (`:string`, allow_nil?: false, public?: true), `create_timestamp(:inserted_at)`, `update_timestamp(:updated_at)`
- Relationships: `belongs_to :channel` (allow_nil?: true), `belongs_to :sender, Astraplex.Accounts.User` (allow_nil?: false). Note: `conversation_id` column NOT added yet -- Phase 5 will add it when Conversation resource exists. Adding a nullable FK column without a resource creates Ash compilation issues.
- Actions: `defaults [:read]`, `:send_message` (create action, accept [:body, :channel_id], change `relate_actor(:sender)`)
- Validations on `:send_message`: require `channel_id` to be present (Phase 4 only sends to channels). Add a custom validation that loads the channel and checks `channel.status != :archived` -- or use a policy expression.
- Policies: `:send_message` allows if `expr(exists(channel.memberships, user_id == ^actor(:id)) and channel.status != :archived)`. Read allows if `expr(exists(channel.memberships, user_id == ^actor(:id)))`. Admin can also read all.
- PubSub notifier: module `AstraplexWeb.Endpoint`, prefix "channel", publish :send_message ["messages", :channel_id]
- Postgres: table "messages", repo Astraplex.Repo
