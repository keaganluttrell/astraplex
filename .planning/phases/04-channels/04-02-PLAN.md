<task type="auto">
  <name>Task 2: Build admin channel management LiveView with drawers and modals</name>
  <files>lib/astraplex_web/live/admin/channel_list_live.ex, lib/astraplex_web/router.ex</files>
  <action>
Create `AstraplexWeb.Admin.ChannelListLive` following the same patterns as `UserListLive` (admin_shell, AshPhoenix.Form, drawers for forms, modals for destructive actions).

**Router updates** in `lib/astraplex_web/router.ex`:
Add inside the existing admin scope's `ash_authentication_live_session :admin` block:
```elixir
live "/channels", ChannelListLive, :index
live "/channels/:id", ChannelListLive, :show
```
The `:show` action opens the channel settings drawer with member management.

**LiveView structure:**

Mount:
- Load all channels via `Ash.read!(Astraplex.Messaging.Channel, actor: current_user)` sorted by name
- Load all active users via `Ash.read!(Astraplex.Accounts.User, actor: current_user)` for the user picker
- Initialize assigns: channels, users, channel_form (nil), selected_channel (nil), member_search (""), selected_member_ids ([])

Handle_params:
- `:index` action: clear selected_channel, close drawers
- `:show` action with `id` param: load channel with `Ash.get!` including `load: [:members]`, set selected_channel, open settings drawer

Render:
- Wrap in `Layouts.admin_shell` with `active_page={:admin}` and breadcrumb_path `[{"Astraplex", "/"}, {"Admin", "/admin/users"}, {"Channels", nil}]`
- Use `page_header` with title "Channels" and "New Channel" button (opens create drawer)
- DaisyUI table showing: name (with # prefix), description (truncated), status badge (active=success, archived=warning), member count, actions column with settings icon
- If no channels, show `empty_state` with icon="hero-hashtag", title="No channels yet", description="Create your first channel to get started"

**Create/Edit Drawer** (reusable for both):
- DaisyUI `drawer` component, controlled by `show_channel_drawer` assign
- AshPhoenix.Form: `for_create` (new) or `for_update` (edit) on Channel resource
- Fields: name (text input, required), description (textarea, optional)
- Submit button: "Create Channel" or "Save Changes"
- Handle "save_channel" event: submit form via `AshPhoenix.Form.submit`, on success reload channels and close drawer, on error show validation errors (including name uniqueness)

**Settings Drawer** (channel detail + members):
- Opens when navigating to `/admin/channels/:id`
- Sections:
  1. **Details**: name, description fields (editable, same form pattern)
  2. **Members**: member_list component with removable=true, "Add Members" button
  3. **Danger Zone**: "Archive Channel" button (red, opens confirmation modal)
- Save details: same "save_channel" event

**Add Members Flow**:
- "Add Members" button toggles `show_user_picker` assign
- Renders user_picker component with users filtered to exclude existing members
- Search filters users by email (client-side filter on assign or phx-change refilter)
- Toggle event adds/removes user ID from `selected_member_ids`
- "Add Selected" button fires "add_members" event: iterates `selected_member_ids`, calls `Ash.create!(Membership, %{channel_id: channel.id, user_id: uid}, actor: actor)` for each, reloads channel members, closes picker

**Remove Member Flow**:
- Remove icon click sets `confirm_remove_member` assign with user struct
- Confirmation modal: "Remove [email] from #[channel_name]?"
- Confirm fires "remove_member" event: `Ash.destroy!` the membership, reload members

**Archive Flow**:
- "Archive Channel" button sets `confirm_archive` assign to true
- Confirmation modal: "Archive #[channel_name]? This will prevent new messages."
- Confirm fires "archive_channel" event: `Ash.update!(channel, %{}, action: :archive, actor: actor)`, redirect to /admin/channels

**Key patterns from UserListLive to follow:**
- Use `AshPhoenix.Form.for_create/for_update` with `to_form` for form handling
- Use DaisyUI `drawer` for all edit forms
- Use DaisyUI `modal` only for destructive confirmations (archive, remove member)
- Delegate ALL data operations to Ash actions with `actor: socket.assigns.current_user`
- Reload data after mutations by re-reading from Ash

Keep the LiveView under 300 lines. If approaching, extract helper functions or move render sections to function components in the messaging components module.
